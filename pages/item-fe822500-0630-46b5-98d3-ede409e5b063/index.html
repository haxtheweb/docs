The real power of HAX comes in the ability to simplify web creation to match specific use-cases people have for documents on the internet. Much like a templated Google/Word doc, HAX skeletons create use cases in the user's mind to build rapidly. Help us HAX as much of the web as possible by solving these use case examples in our issue queue. 

HAX The Whole Web tagged issues on Github

For each, there's a 5 step plan toward contribution:

  1. Find a use case that you can provide a solution for and research / document the needs
  2. If a theme does not exist to satisfy the use case, then mock one up via prototyping tools or creating a new theme using the HAX tooling.
  3. Identify blocks currently lacking in the ecosystem as well as blocks that should be there for this use case
  4. Create a site with content mocked up to match the use case as best as you can
  5. Download the skeleton of the site and submit it to the issue queue

Using this 5 step process, you can help the team HAX The Whole Web. We want to make web publishing feel like editing a .doc file, be portable like a doc, but easy to start, edit, remix, move, and share.
