HAX supports a broad set of formats for authoring, import workflows, machine-readable output, and AI discoverability. This page is the canonical list of format support across HAX and HAXcms.

Authoring and import formats supported in HAX

  • HTML / Web Components - Canonical authoring format used for all saved page content.
  • Markdown (.md) - Can be authored, imported, and converted to HTML.
  • MathJAX / LaTeX - Mathematical notation support in page content.
  • DOCX, PPTX, XLSX - Supported for import into site structure (outline / page generation) and for converting content directly into pages.
  • PDF - Supported for importing and embedding in content workflows.
  • HTML files / URLs - Import HTML directly from files or remote URLs.
  • Platform / system imports - HAXcms, Notion, GitBook, Pressbooks, WordPress, Drupal, ELMS:LN, and additional ecosystem migration pathways.

File upload types accepted by HAXcms

  • Images: jpg, jpeg, png, gif, webp
  • Video: mp4, webm, mov
  • Audio: mp3
  • Documents / data: csv, txt, rtf, vtt, html, md, pdf, doc, docx, xls, xlsx, ppt, pptx

Note: SVG is currently not part of the supported content format list.

Page-level alternate formats generated on save

Every page save writes multiple machine-friendly representations in addition to canonical page HTML.

  • .html - Canonical rendered page content
  • .md - Markdown version of the page content
  • .json - Structured page payload including metadata and content
  • .yaml - Structured page payload including metadata and content in YAML
  • .xml - Structured page payload including metadata and content in XML

AI discoverability and understanding

HAXcms includes first-class support for AI-oriented discovery metadata as part of its format ecosystem.

  • llms.txt is generated from your manifest and page outputs to provide LLMs with a clean site map plus machine-readable content links.
  • robots.txt includes discoverability directives, including Sitemap and LLMS references, so AI systems and search crawlers can locate the right resources.
  • Generated page format variants (.md, .json, .yaml, .xml) improve machine understanding and downstream tooling interoperability.

For any page route, append an extension to access the alternate format directly. Example for this page: /documentation/data-formats.md, /documentation/data-formats.json, /documentation/data-formats.yaml, and /documentation/data-formats.xml.

HAXcms also supports content negotiation via Accept headers for markdown, JSON, YAML, and XML responses, and publishes alternate format links for crawler and client discovery.

Site-level machine-readable and discovery formats

  • site.json - Canonical JSON Outline Schema site manifest
  • rss.xml and atom.xml - Feed formats generated from site content
  • sitemap.xml and sitemap-index.xml - URL discovery maps for search engines and crawlers
  • lunrSearchIndex.json - Search index used by built-in site search
  • robots.txt - Crawl rules and discoverability directives
  • llms.txt - LLM-oriented site map for AI understanding and retrieval
