The HAX changelog tracks major updates shipped across the ecosystem. Use this page for a quick summary view, then follow the linked release feeds for full details.

Where to track releases

  • webcomponents releases for elements, themes, and DDD updates.
  • create CLI releases for authoring workflow and scaffolding changes.
  • haxcms-nodejs releases and haxcms-php releases for backend and endpoint updates.
  • open-apis releases for import, conversion, and service-layer updates.

Suggested upgrade workflow

  1. Review release notes for breaking changes and dependency expectations.
  2. Apply updates in a staging site and test your most-used authoring and import paths.
  3. Run Reports-based checks (especially link checks) after upgrades.
  4. Publish once page rendering, media, and navigation all validate.

If you want a release callout highlighted here, open an item in the unified issue queue.
